WEBDRIVER BIDI VS PLAYWRIGHT
THE FUTURE OF TEST AUTOMATION
Whether you're choosing tools today or planning for tomorrow, here's what you need to know
WHAT IS WEBDRIVER BIDI?
  • Two-way communication protocol (not walkie-talkie like old WebDriver)
  • Real-time events: JS errors, network requests, console messages
  • W3C standard backed by all major browser vendors
CURRENT IMPLEMENTATION
  • Firefox: ~85% ready (3382/3993 tests passing)
  • Chromium: In progress with good coverage
  • WebKit/Safari: Early stages, catching up
2025 REALITY CHECK
PLAYWRIGHT TODAY
  • Complete framework with built-in tools
  • Enhanced browsers with extra hooks
  • ~85% BiDi tests already passing
SELENIUM + BIDI
  • Partial BiDi support since v4
  • Still depends on CDP for advanced features
  • Future promise of cross-browser parity
The future isn't Playwright vs BiDi - it's BiDi powering both,
with Playwright leading on developer experience
